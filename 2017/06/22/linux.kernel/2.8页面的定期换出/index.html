2.8 页面的定期换出

  • Kernel
  • Linux
Kernel

2.8.1 场景类比

场景类比，Linux系统这里对页面的处理比较绕，可能采用场景类比的思想，可以更加方便。

计算机 保险服务
CPU 柜员
Memory 柜台空间
内存页内容 一个一个的保险单
进程 客户
进程请求的页面 客户要的不同类型的证明等等
多个进程同时运行 柜员同时服务多个客户

2.8.2 换出页面场景

对于计算机而言，换入换出的含义如下：

操作 含义
页面换入 从磁盘上读入内容，加载到内存中
页面换出 把页面上的内容同步到磁盘上

那么对应到保险行业的场景可以描述如下：

操作 含义
换入 去仓库里面找到客户的资料，证明，保险单等材料
换出 把客户的各种资料，证明，保险单等材料放到仓库

2.8.3 为什么需要定期换出？

为啥要换来换去的？

是因为柜台空间有限。如果你一直把客户的资料放在那里。新来的客户时候容易发生：

  • 柜台空间不够了，档住了客户。不能提供保险服务了
  • 让客户等一会，把柜台上的资料放到仓库里面。再接客户的请求。

这两种都是不太理想的。所以，定期换出，相当于定期清理柜台桌面。让客户来的时候，总是有空间提供服务。当然，也是有缺点的。比如刚把A客户的资料送到仓库里面去，结果A客户又回来继续办理业务了，这个时候还得去仓库里面把资料取回来。尽管有这样的缺点，但是这只是一个概率问题。就是假设当A客户离开一段时间之后，回来再办理相同业务的可能性比较小。

定期换出目的，对于保险这个场景来说，很明确。就是想办法把柜员的桌面清理一下。不常用的资料放到仓库里面去，留出空间来接新的客户。

2.8.4 内核进程与客户进程

分类 含义
内核进程 柜员在盖章，搬资料，开证明，提供服务
客户进程 客户填写相应的表单，客户自己跑路去别的地方开证明

专门负责定时换出的进程名为kswapd。与客户进程相同的地方：

  • 也是具有task_struct结构
  • 也接受内核的调度

不一样的地方在于，kswapd是一个内核线程，里面对应的地址空间是内核。所以kswapd并不需要通过系统调用来使用内核的功能。

如果高水位的内存数量比当前可用内存数量多，说明现在可用的已经低于freepages.high了。
应该是需要启动页面交换了。

同步还是异步

需要注意的是，什么时候用同步，什么时候用异步？这需要看内存页面是否够用。简单点说，就是：

  • 当内存页面紧张的时候，用同步。因为一波人等着用内存呢，赶快想点办法挤点出来吧。
  • 当内存还有点存量，虽然也不太足够的时候，用异步。异步就是把刷写页面的请求提交出去
    后面有机会再把页面刷写出去。

launder_loop

这是第几次刷写页面了。

  • 第一次主要是移动页面，代价比较小。
  • 第二次是开始刷写页面，代价比较大。并且看情况来决定是否会执行。

maxlaunder

首先看一下，哪些地方使用到了这个变量：

需要注意的是，sync变量在这里起的作用是有点意思的。比如考虑sync = 1的场景。

  • 第一轮扫描页面，没得说，肯定都是移动页面到其他list为主。
  • 第二轮扫描页面，当maxlaunder大于0的时候，采用异步IO
  • 第二轮扫描页面，当maxlaunder用光，并且sync为1的时候，表明可以同步等待页面刷写。
    那么就同步刷写页面。

而当sync = 0的时候，

  • 第一轮扫描页面，没得说，肯定都是移动页面到其他list为主。
  • 第二轮扫描页面，当maxlaunder大于0的时候，采用异步IO
  • 第二轮扫描页面，当maxlaunder用光，并且sync为0的时候，wait = 0，这个时候行为
    回退到第一轮扫描的情况。又只能移动页面到各个list。

cleaned_pages

这里看一下，什么情况下cleaned_pages会增加。

  • 当刷写页面，并且释放缓存之后。如果发现这个页面并没有设备上的页面与之映射，则直接变成可用页面。
  • 有设备与之映射，但是只有一个进程在引用，即kswapd，那么直接可以放到可用页面中。
  • 如果有设备映射，但是页面是干净的。那么直接也是可用页面。
  • 当需要展开第二轮扫描的时候，如果发现cleaned_pages已经刷写了一些页面出去，那么第二轮扫描
    就不需要再采用同步刷写的方式了。实际上的意思是说，我已经腾出一些页面了。可能不是那么紧张了。
